# Research Workflow

Use this file when the request depends on current Tomato Novel platform observations gathered through browser tools.

## Goal

Use browser evidence from the Chinese web interface to answer questions such as:

- what directions currently appear frequently
- what kinds of titles seem common
- what blurb hooks are repeated
- how opening chapters try to hold retention

## Sampling Procedure

1. Start from one ranking page or category page
2. Sample at least 8-15 visible books when possible
3. Record repeated structures instead of isolated examples
4. Compare more than one page type when useful

Good comparison pairs:

- ranking page vs category page
- male-targeted section vs female-targeted section
- new book page vs hot book page
- multiple book detail pages with visible blurbs

## What To Extract

For each sample batch, look for:

- title length and rhythm
- whether titles emphasize identity, reversal, profession, rebirth, system, crisis, romance tension, or status gain
- blurb opening sentence style
- whether the blurb promises power fantasy, comeback, conflict, suspense, emotional payoff, or relationship tension
- how fast the opening setup moves
- where the first strong hook appears

## Research Heuristics

- Favor repeated structures over personal taste
- Separate “high frequency” from “worth using”
- Distinguish category-specific patterns from platform-wide patterns
- If the user has a genre target, bias the sampling toward that genre

## Turn Research Into Decisions

After browsing, convert observations into:

1. 3-5 writing principles
2. 3 original concept directions
3. 3 title formulas
4. 1 recommended opening strategy

## Avoid

- copying exact titles
- paraphrasing a single book too closely
- assuming one visible hit defines the whole platform
- writing a trend report with no recommendation
